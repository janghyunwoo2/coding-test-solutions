# [프로그래머스 - Lv 2] 방문 길이

- **[문제 링크] https://school.programmers.co.kr/learn/courses/30/lessons/49994**
- **날짜:** 2025.11.03
- **사용한 언어:** Python

## 🔍 문제 이해하기
  방향을 입력 받아서 처음 가본 길의 길이 출력
* 방향 9개 받았는데 7개가 처음 가본길이고 나머지 방향 2개가 지나간 길을 갔다면 무효. 결과는 처음 가본길만 체크해서 7


## 💡 핵심 아이디어 / 접근 방식

1.  **문제 이해 및 분석**:
    *   입력
        > 문자열 : ULURRDLLU (U:위 D:아래 R:오른쪽 L:왼쪽) 500이하
    *   출력
        > 숫자 : 7
    *   예외
        > 좌표평면(-5x5) 을 넘어가는 방향은 무시   
        > 가봤던 길 무시
    * 분석 요약
        > 지문 자체는 이해하기 쉬움. 좌표평면이랑 방향을 어떻게 구현 할지..
2.  **알고리즘 설계**:
    *   **아이디어 1**
        > 좌표평면 구현을 위한 0으로 초기화된 2차원 배열을 생성  
        > 지나갈 때 마다 1로 카운팅하기  
        > 이미 지나간 길(1로 카운팅 된 곳), 좌표평면 벗어나면(인덱스 아웃) 무시
        *   시간 복잡도: `O(N)` 
        *   공간 복잡도: `O(N^2)` 
    *   **아이디어 2**
        > 평면 좌표를 생성하지 말고, 이전,이후 좌표 포인터만 기억하기
        > 그리고 set으로 중복 제거
        *   시간 복잡도: `O(N)` 
        *   공간 복잡도: `O(N)` 
    
    
3.  **코드 구현 및 테스트& 디버깅**:
    - [코드 구현](./solution.ipynb)

4.  **피드백**:
    * 아이디어1처럼 구현하려고 하니 새로운 길인데 지나갔던 포인트여서 중복된 길로 체크되기 때문에 다른 방법이 필요했음. 그래서 아이디어 2처럼 구현했음
    * set으로 중복 제거하는 것까지는 좋았음. 다만 왔던길을 되돌아가는 a->b a<-b 를 구현을 못했음. 일일히 하나씩 비교해서 하면 가능하겠다만.. 더티코드가 되어버려서 포기
    * 정답지보니까 이전,현제 좌표 기록 할 때 정렬을 하고 다시 튜플로 바꾸면 구현 할 수 있었음.. 간단했는데 아쉽다.
    > 배울점
    -  좌표 이동을 딕셔너리로 구현해서 중복 코드를 가능한 한 없앰
